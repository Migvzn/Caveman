# 09 — EXPÉRIENCE UTILISATEUR COMPLÈTE

## Principes UX
1. **Une phrase suffit** — l'utilisateur ne devrait jamais remplir un formulaire avant d'avoir vu un résultat.
2. **Valeur avant friction** — pas d'inscription tant qu'on n'a rien à proposer.
3. **Visualiser plutôt que lister** — carte + timeline > tableau.
4. **Réversibilité** — toute modification est annulable (Cmd+Z fonctionne partout).
5. **Honnêteté radicale** — afficher TOUS les coûts, frais inclus, dès la première vue.
6. **Vitesse perçue** — streaming, skeleton, animations courtes.

## Parcours utilisateur principal (happy path)

```
Découverte → Phrase libre → Streaming résultat → Sélection scénario
   →  Édition itinéraire  →  Sauvegarde (+ auth)  →  Réservation
   →  Pendant le voyage  →  Carnet post-voyage  →  Reco voyage suivant
```

### Étape par étape

#### 1. Découverte (landing)
- Démo vidéo 12s en boucle silencieuse.
- Champ "Décris ton voyage" focus auto.
- Exemples cliquables qui pré-remplissent.

#### 2. Phrase libre (5-15s utilisateur)
- L'utilisateur tape ou clique un exemple.
- Bouton Enter ou ↑ envoie.

#### 3. Streaming résultat (≤ 60s perçu)
- **0-1s** : "J'ai compris : Lisbonne, 5 jours, juin, 800€ depuis Lyon. ✓" (validation visible)
- **1-2s** : peut-être 1 question (boutons rapides)
- **2-12s** : "Je compare 47 vols et 23 hôtels..." avec compteurs qui montent
- **12-20s** : 3 cartes scénarios apparaissent une par une avec stagger
- **20-60s** : itinéraire détaillé se rend progressivement

#### 4. Sélection scénario
- Tap sur une carte → animation d'expansion fluide.
- Toggle entre les 3 scénarios sans recharger (préchargés).

#### 5. Édition itinéraire
- Drag-and-drop activités (intra-jour ET inter-jour).
- Tap "+" pour ajouter (autocomplete POIs Google Places).
- Suppression : swipe gauche mobile, X bouton desktop.
- Recalcul auto trajets+prix à chaque changement (debounced 800 ms).
- Indicateur "non sauvegardé" en haut.

#### 6. Sauvegarde + Auth (juste à temps)
- "Sauvegarde ton voyage" en sticky bar → demande Google/Apple/Email.
- L'état est conservé pendant l'auth (pas de perte).

#### 7. Réservation
- V1 : redirection séquentielle vers fournisseurs avec deep-link affiliés, contexte conservé en local storage.
- V2 : checkout unifié Stripe avec une seule signature CB.
- Confirmation : email + push + "Mes billets" centralisés.

#### 8. Pendant le voyage (V1.1)
- Carte hors ligne pré-téléchargée.
- "Aujourd'hui" widget avec activités du jour.
- Re-planification en cas d'imprévu : "Mon train est annulé" → l'IA propose alternatives.

#### 9. Post-voyage (V2)
- "Notation" rapide (slider de 1 à 5 sur 3 axes) + 1 photo.
- IA condense en faits utilisateur ("aime Lisbonne", "auberges trop bruyantes").
- Suggestion voyage suivant basée sur historique.

## Design System

### Tokens
| Token | Valeur | Usage |
|---|---|---|
| `--color-bg` | #0B0B0F (dark) / #FAFAFA (light) | fond app |
| `--color-surface` | #14141A / #FFFFFF | cartes |
| `--color-fg` | #F5F5F7 / #0B0B0F | texte primaire |
| `--color-muted` | #8B8B95 | texte secondaire |
| `--color-accent` | #FF5C39 (orange brûlé) | CTA |
| `--color-accent-2` | #5D6BFF (bleu) | liens, icones |
| `--color-success` | #2EBD7A | confirmations |
| `--color-warning` | #FFC857 | alertes |
| `--color-danger` | #E5484D | erreurs |
| `--radius-sm` | 6px | inputs |
| `--radius-md` | 12px | cartes |
| `--radius-lg` | 20px | modales |
| `--radius-pill` | 999px | badges |

### Typographie
- **Titres** : Inter Display ou Söhne Breit, weights 600/700
- **Texte** : Inter, 400/500
- **Mono** : JetBrains Mono (prix, dates dans certains contextes)
- **Échelle** : 12 / 14 / 16 / 18 / 22 / 28 / 36 / 48

### Spacing
- Base 4 px : 4, 8, 12, 16, 24, 32, 48, 64, 96.

### Animations
- **Durée** : 120 ms (micro), 240 ms (transitions), 400 ms (page).
- **Easing** : `cubic-bezier(.2, .8, .2, 1)` (out-quint) pour entrer ; `cubic-bezier(.4, 0, 1, 1)` pour sortir.
- **Stagger** : 40 ms entre items d'une liste qui apparaît.

### Composants signature

#### Composer chat
- Hauteur dynamique, max 6 lignes.
- Suggestions inline (chips au-dessus).
- État loading : barre de progression sous le champ pendant streaming.

#### Carte scénario
- Hauteur fixe ; en haut un badge (ECO / ÉQUILIBRÉ / CONFORT) avec icône.
- Prix gros, breakdown en mini-bar horizontale.
- "Inclus" : 3 icônes (transport / hôtel / activités) avec micro-photo.

#### Timeline jour
- Sticky header de date au scroll.
- Items connectés par une ligne verticale tenue (comme une vraie timeline).
- Heure à gauche, contenu à droite, prix en bas à droite.

#### Map view
- Mapbox style "monochrome" custom (sobre, contrastes nets).
- Pins colorés par jour (jour 1 = orange, jour 2 = bleu, etc.).
- Bottom sheet rétractable.

### Iconographie
- **Lucide React** comme base (cohérence, gratuit, large set).
- Icônes custom pour transport mixte (avion + train).

### Dark mode
- Premier-class citizen, switch dans header.
- Respect `prefers-color-scheme`.
- Couleurs ajustées (pas juste inversion) — la carte Mapbox a un style dédié.

### Accessibilité
- **WCAG 2.2 AA** minimum.
- Contraste AAA sur les CTAs.
- Navigation clavier complète.
- Annonces ARIA pour les états de chat (streaming, scénarios prêts).
- Respect `prefers-reduced-motion` (désactive animations longues, garde feedback).
- Support lecteur d'écran sur la carte (liste alternative).

### Internationalisation
- FR + EN au lancement.
- Tout texte via `next-intl`.
- Dates/devises selon locale.
- RTL-ready (arabe en V2).

## Anti-patterns à éviter
- Pas de modal lourde au premier chargement (pas de cookie banner intrusif — bandeau bottom-sticky minimal).
- Pas de pop-up "abonnez-vous à la newsletter".
- Pas de prix annoncés bas qui montent au checkout.
- Pas de bouton "Réserver" qui mène à un formulaire de 5 étapes — V2 = 1 écran.
- Pas de skeleton qui dure > 3s sans message.

## UX writing
- Ton : direct, complice, jamais commercial.
- Verbes à l'impératif sur les CTAs ("Lancer", "Réserver", "Modifier").
- Erreurs : courtes, expliquent ce qui s'est passé, proposent une action.
- Émojis : interdits dans l'interface (sauf input utilisateur).

## Tests UX
- **5-second test** : nouveaux utilisateurs comprennent en 5s que c'est un planificateur IA.
- **A/B** sur la landing dès J+30.
- **Tests modérés** sur 8 utilisateurs avant chaque release majeure.
- **Session replays** (PostHog) anonymisés.
